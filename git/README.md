# Git

## Создание репозитория

`git init` - инициализация репозитория

`git clone [url]` - клонирование репозитория

`git clone [url] [folder-path]` - клонирование репозитория в папку

## Запись изменений в репозиторий

`git status` - проверка состояния файлов

`git status -s` - проверка состояния файлов (краткий отчет)

`git add [folder|file]` - слежение за новыми файлами

`git diff` - просмотр индексированных и неиндексированных изменений

`git diff --staged` - просмотр того, что войдет в следующий коммит

`git diff --cached` - просмотр проиндексированных изменений

`git commit -m 'First commit'` - фиксация изменений (коммит с сообщением) 

`git commit -a` - фиксация изменений с автоматической индексацией отслеживаемых файлов

`git rm [file]` - удаление файла из области индексирования

`git rm -f [file]` - удаление проиндексированного файла из области индексирования
 
`git rm --cached [file|pattern]` - удаление файла из репозитория, но не из рабочей папки

`git mv [old-file-name] [new-file-name]` - переименовывание файла

## Просмотр истории версий

`git log` - просмотр истории версий

`git log -p` - просмотр истории версий и разницей в каждом коммите

`git log -2` - просмотр истории версий последних 2 коммитов

`git log --stat` - краткая статистика по каждой версии

`git log --pretty` - выводит информацию в другом формате

`git log --pretty=[oneline|short|full|fuller]` - выводит информацию в разных форматах

`git log --pretty=format:"%h - %an, %ar : %s"` - выводит информацию в разных форматах по шаблону ([Полная документация по шаблонам](https://git-scm.com/docs/pretty-formats))

`git log --pretty=format:"%h %s" --graph` - полезный флаг `graph` выводит информацию в виде графа

`git log --since=2.weeks` - история за последние 2 недели

`git log --author=[name]` - поиск по автору

## Отмена изменений

`git commit -amend` - отмена изменений последнего коммита

## Отмена индексирования

`git reset HEAD [file]` - отмена индексирования

## Работа с удаленными репозиториями

`git clone [url]` - клонирование репозитория

`git remote` - работа с удаленными репозиториями

`git remote -v` - позволяет увидеть URL и сокращенное имя

`git remote add [short-name] [url]` - добавление удаленного репозитория

`git fetch [name]` - скачивание информации с определенного сервера

`git pull` - извлекает файлы с сервера с которого был сделан `clone`

`git push [nameServer] [branch]` - отправка данных на удаленный сервер

`git remote show origin` - дополнительная информация об удаленном репозитории

`git remote rename [oldName] [newName]` - переименовывание удаленных репозиториев

`git remote rm [name]` - удаление удаленного репозитория

## Теги

`git tag` - выводит список тегов

`git tag -l ,[pattern]` - выводит список тегов с фильтром по паттерну

`git tag -a [name] -m '[message]'` - добавление тега с комментарием

`git tag show [tagName]` - просмотр данных тега с помеченным коммитом

`git tag [name]` - создание легковесного тега

`git push origin [tagName]` - отправка тега на удаленный сервер

`git push origin --tags` - отправка на удаленный сервер всех тегов

## Создание псевдонимов (Alias)

`git config --global alias.[aliasName] [command]` - создание алиаса

## Работа с ветками

`git branch [name]` - создание новой ветки

`git checkout -b [name]` - создание новой ветки и переход на нее

`git checkout [name]` - переход на другую ветку

`git merge [branchName]` - объединение выбранной ветки с текущей

`git branch -d [branchName]` - удаление ветки

`git branch -v` - показывает последний коммит в каждой ветке

`git branch --merged` - слитые с текущей веткой ветки

`git branch --no-merged` - не слитые с текущей веткой ветки
